## Домашнее задание №4 «Сохранение данных в БД»

1. Добавить миграции на создание таблиц в БД.

   БД должна быть реализована в 3 нормальной форме.

2. Подключить работу с базой данных, добавить конфиг подключения, инициализацию коннекта.
   
3. Написать CRUD-операции для работы с БД.

   > Подсказка, вам могут пригодиться следующие методы:
   > * GetByID
   > * List
   > * Update
   > * Create
   > * Delete

Также необходимо реализовать те методы, которые вы делали с файлом.
Поведение для пользователя приложения не должно измениться, только вместо файлов теперь используется БД.

4. Добавить новые API-ручки: 

   4.1. **Получение истории по заказу**

   На вход принимает ID заказа, на выходе - массив с историей статусов по заказу, упорядоченный по уменьшению времени изменения.

   Если такого заказа нет - выдает ошибку.

   *Подробнее смотри в файле contracts.md*

   > Не забудьте перегенерировать swagger-спецификацию.

> Не забудьте проверить с помощью EXPLAIN план выполнения запросов и добавить необходимые индексы.

### Ограничения

* Нельзя использовать ORM или SQL билдеры

### Дополнительные задания

Необходимо реализовать клиент для работы с БД, который:
1. будет логировать запросы к базе
2. предоставит возможность выполнять запрос внутри транзакции
3. позволит выбрать режим подключения (read/write), как будто у нас есть несколько реплик БД

    > Подсказка: клиент-обертку над БД лучше разместить в pkg.
    Репозитории - в internal.

High-level:
4. прикрутить реальную master-slave репликацию к БД
